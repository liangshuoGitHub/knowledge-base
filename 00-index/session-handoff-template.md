# 会话收尾沉淀模板

## 这页是干什么的

实操时，一个主题常常被拆进好几个会话。每个会话快结束时，里面聊出来的结论、命令、踩的坑，如果不收一下，会话一关就散了。

**现在收尾入库已经自动化了**：会话里随时说一句「沉淀知识库草稿」，全局 skill `knowledge-base-deposit` 会基于当前会话上下文，自动生成草稿并带时间戳写进统一收件箱 [`inbox.md`](./inbox.md)，你不用再手动复制提示词、也不用手动粘贴。

这一页保留下来，主要是记录**草稿应该长什么样**，方便你了解 skill 生成的结构，或在没有 skill 的环境里手动让模型照着这个结构整理。整个流程的位置见 [`知识库使用方法`](./index.md)。

## 草稿的标准结构

skill 自动生成的草稿（以及你手动整理时应遵循的结构）长这样：

```
## [写入时间戳] 草稿主题标题

- 主题：（这次主要在聊什么，一句话）
- 关联仓库/项目：（如果涉及真实项目，写明是哪个仓库，没有就写无）

### 结论 / 认知
（这次想明白的关键结论，讲人话，能复述）

### 命令 / 代码片段
（高频命令、关键代码，用代码块；没有就省略）

### 术语
（这次出现、以后可能会忘的名词，给通俗解释；没有就省略）

### 待验证问题
（还没想透、需要以后确认的点；没有就省略）
```

要求：

1. 讲人话，别写成教科书
2. 只留真正值得记的，别为了凑字数堆细节
3. 代码用代码块，单个名词用行内代码
4. 用简体中文

## 没有 skill 时的手动办法（备用）

如果在一个没接全局 skill 的环境里，也可以手动收尾：把下面这段话发给当时的模型，让它照结构吐出草稿，再自己贴进知识库的 [`inbox.md`](./inbox.md)。

```
我们这个会话要结束了。请把本次值得沉淀进我个人知识库的内容，
整理成一段我能直接复制粘贴的 Markdown 草稿，按下面的结构组织：

## 本次会话沉淀草稿
- 主题：（这次主要在聊什么，一句话）
```

## 小提醒

- 不是每个会话都值得收尾沉淀。只有真聊出了**结论、坑、术语、项目认知**时才有必要
- 草稿是「原料」不是「成品」，先进收件箱，别指望一步到位变正式文档
- 草稿进了 [`inbox.md`](./inbox.md) 之后，在知识库目录下说一句「整理碎片」，就会被提炼进正文对应章节
