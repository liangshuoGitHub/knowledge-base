# 会话收尾沉淀模板

## 这页是干什么的

实操时，一个主题常常被拆进好几个会话。每个会话快结束时，里面聊出来的结论、命令、踩的坑，如果不收一下，会话一关就散了。

这页就是一段**可以直接复制的话**。结束某个会话前，把它发给当时正在对话的模型，让它把本次值得留下的内容，整理成你能直接拿去沉淀的草稿。

整个流程的位置见 [`知识库使用方法`](./index.md)：这一步产出的草稿，下一步会先进对应文档的「待整理碎片」区。

## 收尾提示词（复制这段）

```
我们这个会话要结束了。请把本次值得沉淀进我个人知识库的内容，
整理成一段我能直接复制粘贴的 Markdown 草稿，按下面的结构组织：

## 本次会话沉淀草稿
- 主题：（这次主要在聊什么，一句话）
- 关联仓库/项目：（如果涉及真实项目，写明是哪个仓库，没有就写无）

### 结论 / 认知
（这次想明白的关键结论，讲人话，能复述）

### 命令 / 代码片段
（高频命令、关键代码，用代码块；没有就省略）

### 术语
（这次出现、以后可能会忘的名词，给通俗解释；没有就省略）

### 待验证问题
（还没想透、需要以后确认的点；没有就省略）

要求：
1. 讲人话，别写成教科书
2. 只留真正值得记的，别为了凑字数堆细节
3. 代码用代码块，单个名词用行内代码
4. 用简体中文
```

## 拿到草稿之后怎么办

把模型吐出的草稿复制下来，开一个维护知识库的会话，说一句类似：

> 把这段会话草稿放进待整理碎片区。（粘贴草稿）

剩下的归位、改写、提炼、清空，按知识库的维护流程走，你不用自己手动整理 Markdown。

## 小提醒

- 不是每个会话都值得收尾沉淀。只有真聊出了**结论、坑、术语、项目认知**时才有必要
- 草稿是「原料」不是「成品」，先进碎片区，别指望一步到位变正式文档
- 如果当时偷懒没让模型整理，事后直接把聊天里的关键段落复制出来也行，只是质量会差一点
